# 《专业大逃杀》Steam 上线检查清单

## 使用方式

本清单区分“仓库内可验证”“需要 Steamworks 账号/AppID”“需要发行方签名或人工素材”三类事项。只有阻断项全部关闭，才能把候选包设置为 Steam 默认分支；不能用本地测试通过代替 Valve 审核通过。

官方依据：

- Valve 商店页与构建审核：https://partner.steamgames.com/doc/store/review_process

- Steam 成就接入：https://partner.steamgames.com/doc/features/achievements/ach_guide
- Godot 许可合规：https://docs.godotengine.org/en/stable/about/complying_with_licenses.html
- LXGW WenKai 授权：https://github.com/lxgw/LxgwWenKai/blob/main/OFL.txt

- [x] Godot 4.4.1 对应双平台导出模板可被脚本预检。
- [x] 导出前强制完成资源导入，导出后检查候选包并生成 SHA-256 摘要。
- [x] 在线 AI 默认关闭；离线白名单策略可完成全部 AI Native 战斗。
- [x] 在线 AI 响应限制为 64 KiB，非法、超时、旧响应均安全回退。
- [x] 一局进度使用版本化 JSON、临时文件原子替换和上一版本备份。
- [x] 局外成长使用独立版本化档案，永久金币、已结算局、天赋、装备和强化均原子保存并保留备份。
- [x] 主菜单可继续安全检查点；战斗中崩溃后从该战斗初始状态恢复，不保存不一致的半回合。
- [x] 108 张卡牌均有 256×256 独立插画，五专业与十敌人都有正式立绘。
- [x] 数字种子可复现牌序、事件、奖励、敌人意图和精英词缀；种子与挑战档位进入安全存档。
- [x] 四档挑战的解锁本身不发放永久数值；独立局外成长受金币成本、装配槽与三级上限约束。
- [x] 480 局、五专业、三种操作画像的真实规则矩阵在隔离局外档案后保持原平衡区间。
- [x] 1.2 内容基线与 1.3 局外成长增量均有候选说明、自动化门槛和外部阻断记录。
- [x] 1280×720、16:10、1920×1080 与超宽屏采用等比安全区，无拉伸或控件漂移。
- [x] 键鼠与手柄均可完成主流程，暂停、辅助时长、减少动态和震动开关可用。
- [x] LXGW WenKai 的完整 OFL 1.1 文本与 Godot MIT 声明随候选包提供。
- [ ] Windows 实机从全新 Steam 安装目录完成一局，不依赖项目目录、Python 或本机 AI 服务。
- [ ] macOS 候选包完成 Developer ID 签名、公证与另一台 Mac 的 Gatekeeper 验证。
- [ ] 连续三局真人测试确认无软锁、存档倒退、奖励重复、输入丢失和音频疲劳问题。

## B. Steamworks / AppID 阻断项

- [ ] 取得真实 AppID、Windows depot ID、macOS depot ID 和受限构建账号。
- [ ] 在 Steamworks 后台设置安装目录、双平台 depot、Windows/macOS 启动项。
- [ ] 使用 Steamworks SDK 的 SteamPipe 建立 `app_build_<AppID>.vdf` 和 depot 脚本；凭据不得写入仓库。
- [ ] 首次上传到私有 beta 分支，从 Steam 客户端安装并验证卸载、覆盖更新、离线启动和云同步关闭时的行为。
- [ ] 将 14 个本地成就映射为 Steamworks API 名称；未取得 AppID 前不得虚构 ID 或宣称已接通。
- [ ] 决定是否启用 Steam Cloud；若启用，只同步 `run_save.json`、`meta_progression.json`、对应备份、设置与成就文件，并验证冲突策略。
- [ ] 配置“完全支持控制器”或“部分支持控制器”声明，确保商店能力与实测一致。
- [ ] 请求 Steam Deck 兼容性审核。当前自定义专业名称输入可能需要手动呼出键盘，发布前至少按“可玩”标准如实申报。

## C. 商店页与合规阻断项

- [ ] 准备主胶囊、小胶囊、页眉胶囊、库封面、库徽标和社区图；标题必须可读，不得直接拿概念图代替游戏截图。
- [ ] 准备只包含实际玩法的截图：专业选择、校园探索、普通战、AI Native 战、答辩窗口、奖励构筑和终局。
- [ ] 制作带实际游戏画面的预告片；不得承诺未在首发构建出现的多地图、多人联机或云端 AI 能力。
- [ ] 完成简体中文商店文案、系统需求、控制器支持、语言支持、内容问卷和 AI 生成素材披露。
- [ ] 发布隐私说明：默认离线；只有玩家主动启用并配置在线 AI 服务时，战斗上下文才会发送到该地址。
- [ ] 核对所有美术、音频、字体、商标和生成素材的商业使用权与留档证据。
- [ ] 商店页和候选构建分别点击“准备审核”；Valve 通常需要 3–5 个工作日，应至少预留 7 个工作日处理反馈。
- [ ] 不在游戏内索要、诱导或奖励 Steam 好评；仅用更新公告和社区反馈改进产品。

## D. 上线前最终回归

1. 从 Steam 私有分支全新安装，断网启动。
2. 用手柄新开一局，完成事件、普通战、答辩窗口、奖励、AI Native 离线战和 Boss。
3. 在校园、战斗前、奖励后分别强制退出，验证继续游戏位置和资源不倒退。
4. 切换 1280×720、1920×1080、16:10 与超宽屏，验证留边、字体和焦点。
5. 完成三局并观察内存、帧时间、音频循环与存档体积。
6. 验证 Steam 覆盖层、截图、控制器、成就、语言、离线模式和卸载后用户数据策略。
7. 将候选 BuildID、Git commit、SHA-256、测试设备和已知问题写入发行记录，再设置默认分支。

## 当前结论

仓库内工程已进入可导出候选包阶段，但在真实 AppID、SteamPipe 私有分支、Windows 实机、macOS 签名公证、商店素材与 Valve 审核完成前，不能宣称“已可直接公开上线”。这些项目需要发行方账号、证书或人工素材，不应在代码中伪造。
